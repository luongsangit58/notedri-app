## `android/`

Native Android project generated by Expo. Contains `build.gradle`, `AndroidManifest.xml`, and Java/Kotlin wrappers. Required because `react-native-ble-plx` is a native module and cannot use Expo Go - a custom dev client is needed.

The `patches/` directory contains a `patch-package` diff that fixes a `build.gradle` issue in `react-native-ble-plx`. This patch is applied automatically via the `postinstall` script in `package.json`.

## `src/` - Application Source

### `src/api/`

Thin Axios wrapper functions, one file per backend domain. Each file imports the shared `client` and exports async functions used by the TanStack Query hooks.

```
src/api/
├── client.ts         Shared Axios instance with base URL, interceptors (token + 401 logout)
├── achievements.ts   GET /achievements
├── auth.ts           POST /auth/login, /auth/register, /auth/forgot-password, GET /auth/me
├── dashboard.ts      GET /dashboard
├── fuelTypes.ts      GET /fuel-types
├── gpsTrips.ts       GET/POST/DELETE /gps-trips/*
├── notifications.ts  GET/PATCH /notifications/*
├── obd.ts            POST /obd2/trips, GET /obd2/dtc, POST /obd2/dtc/{id}/resolve
├── odometer.ts       GET/POST/PUT/DELETE /odometer/*
├── profile.ts        GET/PUT /profile, POST /profile/password, POST /profile/avatar,
│                     POST /profile/push-token, POST /profile/export, DELETE /profile
├── refuels.ts        GET/POST/PUT/DELETE /refuels/*
├── reminders.ts      GET/POST/PUT/DELETE /reminders/*
├── services.ts       GET/POST/PUT/DELETE /services/*
├── timeline.ts       GET /timeline
└── vehicles.ts       GET/POST/PUT/DELETE /vehicles/*
```

### `src/hooks/`

TanStack Query hooks, one per domain. Each hook encapsulates `useQuery` and `useMutation` calls for its domain. Screens import only the hooks they need.

```
src/hooks/
├── useDashboard.ts       Queries /dashboard
├── useFuelTypes.ts       Queries /fuel-types (used in AddRefuelScreen)
├── useGpsTrip.ts         Queries and mutations for /gps-trips/*
├── useNotifications.ts   Queries and mutations for /notifications/*
├── useObd.ts             Queries and mutations for /obd2/* (DTC, trips)
├── useOdometer.ts        Queries and mutations for /odometer/*
├── useRefuels.ts         Queries and mutations for /refuels/*
├── useReminders.ts       Queries and mutations for /reminders/*
├── useServices.ts        Queries and mutations for /services/*
├── useTimeline.ts        Queries /timeline (infinite scroll or paginated)
├── useVehicles.ts        Queries and mutations for /vehicles/*
└── useVoiceInput.ts      Wraps expo-speech-recognition (no network)
```

### `src/i18n/`

Custom bilingual i18n implementation using a Zustand store instead of the standard `i18next` provider pattern.

```
src/i18n/
├── index.ts    Zustand i18n store: lang state, t() function, setLang(), SecureStore persistence
├── vi.ts       Vietnamese translation dictionary (default language)
└── en.ts       English translation dictionary
```

Usage in screens:
```typescript
const { t } = useI18nStore();
// then:
t('some.key')
```

## Root Files

| File | Purpose |
|---|---|
| `App.tsx` | Root React component. Wraps app with `NavigationContainer`, `QueryClientProvider`, loads custom fonts. Entry point rendered by `index.ts`. |
| `index.ts` | Calls `registerRootComponent(App)`. Expo entry point referenced in `package.json` `main`. |
| `app.json` | Expo managed config: app name, bundle IDs, version, permissions (BLE, location, camera, microphone, notifications), plugin config for native modules. |
| `eas.json` | EAS Build profiles: `development` (dev client), `preview` (APK), `production` (AAB + Proguard + autoIncrement). |
| `tsconfig.json` | Extends `expo/tsconfig.base`. Path aliases if any are defined here. |
| `AGENTS.md` | Instruction for AI coding agents: always read Expo v56 docs before writing code. |
| `CLAUDE.md` | Imports `@AGENTS.md`. |
| `J16 protocol.pdf` | ELM327 OBD2 interface protocol reference PDF (hardware spec for the BLE adapter). |
